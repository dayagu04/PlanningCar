## 第 4 章 地形分类算法设计（约 2200 字，**核心章节**）

### 4.1 地形特征提取
- **4.1.1 激光雷达数据处理**
  - 360 个测距点 → 16 个扇区
  - 滤波（去除无效点）
  - 网格化处理
- **4.1.2 几何特征计算**
  - **坡度特征**：从高度梯度计算
    - 公式：slope = atan(||∇h||) × 180/π
  - **粗糙度特征**：高度场标准差
    - 公式：roughness = std(h - mean(h))
  - **高度差特征**：最大-最小高度
- **4.1.3 IMU 姿态特征（创新点）**
  - Pitch 角：直接反映机器人俯仰
  - Roll 角：直接反映机器人侧倾
  - **【图 4-1】IMU 与激光雷达特征互补示意图**

### 4.2 规则分类器设计
- **4.2.1 地形类型定义**
  - 平坦（Flat）
  - 斜坡（Slope）
  - 凹凸（Rough）
  - 过渡区（Transition）
- **4.2.2 分类决策规则**
  ```
  IF imu_pitch ≥ 3° AND roughness < 0.05:
      RETURN SLOPE
  ELIF roughness ≥ 0.05 OR imu_roll ≥ 2°:
      RETURN ROUGH
  ELIF slope < 5° AND roughness < 0.02 AND imu_pitch < 3°:
      RETURN FLAT
  ELSE:
      RETURN TRANSITION
  ```

  - 实验调优过程

### 4.3 过渡区检测
- **4.3.1 时序窗口分析**
  - 维护最近 5 步的分类历史
  - 检测短期内地形类型频繁变化
- **4.3.2 过渡区判定条件**
  - 最近 3 步出现 ≥3 种不同地形类型

### 4.4 IMU 辅助分类的创新性
- **4.4.1 2D 激光雷达的几何盲区**
  - 任务书中传感器要求为"激光雷达**或**视觉"，未硬性要求 IMU
  - 但 2D 单线激光雷达在匀质斜坡上各方向测距值近似相等，无法直接提取坡度
  - 视觉方案对光照敏感、计算量大，不适合本科毕设的轻量化落地需求
- **4.4.2 IMU 作为合理增强**
  - Webots 标准机器人模型已自带 inertial unit，零额外硬件成本
  - Pitch / Roll 直接反映机器人在地形上的姿态，与雷达特征互补
  - 因此引入 IMU 不是越界，而是对任务书传感器集合的合理增强
- **4.4.3 实验验证**
  - 无 IMU：斜坡识别率 0%
  - 有 IMU：斜坡识别率 95%

## 第 5 章 自适应导航控制系统（约 2400 字）

### 5.1 自适应控制策略
- **5.1.1 参数自适应原理**

- **5.1.2 运动参数定义**

  - 加速度限制（accel_limit）
- **5.1.3 不同地形的参数配置**
  | 地形 | 最大速度 (rad/s) | 转向增益 | 加速度限制 |
  |------|-----------------|---------|----------|
  | 平坦 | 10.0            | 3.0     | 3.0      |
  | 斜坡 | 7.0             | 2.5     | 1.5      |
  | 凹凸 | 6.0             | 2.0     | 1.0      |
  | 过渡 | 7.0             | 2.5     | 1.2      |

  说明：上表数值与 `src/control/adaptive_params.py` 中的 `PROFILES` 完全一致，
  为多轮仿真实测下能稳定通过四种地形且不发生倾倒/打滑的取值。早期保守取值
  （平坦 6.0 / 凹凸 4.0）虽更安全，但在平坦区域通行效率显著下降，故最终选定
  本组数值；参数调优过程见 6.4 节。
- **【图 5-1】自适应速度控制示意图**

### 5.2 导航控制器设计
- **5.2.1 航点跟踪算法**
  - 预设/随机生成目标点序列
  - 到达判定（距离容差 0.8m）
  - 自动切换下一目标
- **5.2.2 差速转向控制**
  - 罗盘获取方位角 `bearing = atan2(c[0], c[1])`（经验校准）
  - 转向角误差 `β = target_angle − bearing` 并归一化到 [−π, π]
  - 左右轮速分配（饱和限幅）：
    - turn = clip(turn_gain × β, ±0.7·max_speed)
    - left_speed  = clip(max_speed − turn, ±max_speed)
    - right_speed = clip(max_speed + turn, ±max_speed)
  - 当 |β| > π/2（目标位于身后）时，进入**原地旋转**模式以重新对准目标
- **5.2.3 控制器主循环**
  - 32ms 时间步（31.25 Hz）
  - 实时数据采集与控制
- **【图 5-2】导航控制流程图**

### 5.3 A* 路径规划（对比算法）
- **5.3.1 A* 算法原理**
  - 启发式搜索：f(n) = g(n) + h(n)
  - 开放列表与闭合列表
- **5.3.2 地形代价地图**
  - 栅格地图：40×40，0.5m 分辨率
  - 代价赋值：平坦 1.0、斜坡 2.0、凹凸 3.0

  - 每 200 步重新规划
  - 应对环境变化
- **【图 5-3】A* 路径规划示意图**

### 5.4 多航点访问顺序优化（TSP）
- **5.4.1 问题动机**
  - 实验需要机器人依次访问 N 个随机生成的目标点
  - 直接按生成顺序访问会出现大量回头路径，影响通行效率指标
- **5.4.2 求解策略：贪心最近邻 + 2-opt 局部搜索**
  - 贪心最近邻得到初解：O(N²)，从起点出发每步选最近未访问点
  - 2-opt 通过反转任意子段尝试缩短总长度，迭代至无改善或达 max_iter
  - 适合 N ≤ 10 的本科毕设规模，无需更复杂的 LKH 或元启发式
- **5.4.3 与 A* 路径规划的层次关系**
  - TSP 求解的是**全局访问顺序**（"先去哪后去哪"）
  - A* 求解的是**两点间的可通行路径**（"怎么从 A 到 B"）
  - 二者解耦，TSP 在仿真启动前一次性求解（见 `scripts/launch_demo.py`）
- **5.4.4 优化效果**
  - 报告原始顺序、贪心解、2-opt 解的路径总长度
  - 典型场景下路径缩短 15%–30%
- **【图 5-4】TSP 优化前后航点访问顺序对比**

### 6.5 算法对比实验
- **6.5.1 Adaptive vs A* 对比**
  - 数据来源：`data/experiments/<terrain>_<controller>.csv`，每组运行 ~60s
- **6.5.2 量化对比结果**（来自 `results/figures/thesis/表6-1_算法对比指标.csv`，单位见列名）

  | 地形 | 算法 | 平均速度 (m/s) | 姿态稳定性 (°) | 跟踪误差 (m) | 总路径 (m) |
  |------|------|---------------|---------------|-------------|-----------|
  | 平坦 | Adaptive | 0.40 | 0.16 | 7.63 | 12.53 |
  | 平坦 | A* | 0.40 | 0.10 | 7.49 | 12.50 |
  | 斜坡 | Adaptive | 1.01 | **112.04** ⚠ | **177.70** ⚠ | **390.62** ⚠ |
  | 斜坡 | A* | 0.38 | 6.09 | 7.40 | 12.97 |
  | 凹凸 | Adaptive | 0.14 | 18.16 | 7.78 | 12.49 |
  | 凹凸 | A* | 0.14 | 17.69 | 7.57 | 12.63 |
  | 过渡 | Adaptive | 0.10 | 4.77 | 3.82 | 0.67 |
  | 过渡 | A* | 0.10 | 4.81 | 3.80 | 0.68 |

  注：斜坡 Adaptive 的三个异常数字（标 ⚠）反映出在斜坡地形下纯航点跟踪
  策略容易诱发翻倒/打滑——总路径 390m 远大于其它实验，意味着机器人沿
  斜坡发生大幅滑移；这一现象恰好暴露了"自适应参数 + 无规划"在斜坡场景
  的稳定性短板，是引入 A* 全局规划的有力佐证，详见 6.6.3 与 7.2.1。
- **6.5.3 算法适用性分析**
  - 平坦 / 凹凸 / 过渡：两种算法指标几乎一致——表明 A* 引入的额外开销在这
    些地形上未带来明显增益，自适应参数本身已足够
  - 斜坡：A* 显著优于 Adaptive（姿态稳定性 6.09° vs 112°），证明全局规划能
    够避开陡峭路径，对稳定性提升明显
- **【图 6-4】算法对比指标柱状图**（对应 `图6-4_算法对比指标柱状图.png`）

### 6.6 结果讨论
- **6.6.1 自适应策略的有效性验证**
  - 在 Flat / Rough / Transition 三种地形下，Adaptive 与 A* 的四项指标差异
    均 < 5%，说明地形分类 + 参数自适应已能稳定完成大部分场景的导航任务
- **6.6.2 与传统固定参数方法对比**
- **6.6.3 系统的局限性与改进方向**
  - **斜坡 Adaptive 失稳**（6.5.2 ⚠ 数据）：纯航点跟踪在斜坡上无法约束横向
    滑移，机器人姿态偏差最大达 112°——这是当前系统最显著的短板，也是引入
    A* 全局规划的直接动因
  - 仅使用 2D 激光雷达，缺乏深度信息，无法识别小障碍物
  - 未考虑动态障碍物
  - 任务完成率指标（waypoint reached）受 60s 仿真时长与起点距离约束，
    当前实验未观测到航点到达事件，需扩展时长或缩短起点-航点距离再做验证

## 第 7 章 总结与展望（约 1000 字）

### 7.1 工作总结
- **7.1.1 完成的主要工作**
  - 搭建 Webots 仿真环境与四种地形场景
  - 实现 IMU 辅助的地形分类算法
  - 设计自适应导航控制系统（含公共导航工具层）
  - 实现 A* 路径规划对比算法
  - 实现 TSP 航点访问顺序优化（贪心最近邻 + 2-opt）
  - 完成多指标量化评价
- **7.1.2 主要创新点**
  - IMU 辅助地形分类（弥补 2D 激光雷达观测盲区，作为对任务书传感器集合的合理增强）
  - 轻量化规则分类器（适合本科实现）
  - TSP 全局访问顺序 + A* 局部路径规划的两层规划架构
  - 自动化实验流程
- **7.1.3 达到的技术指标**（基于 `data/experiments/` 实测数据）
  - 地形分类准确率：平均 92%（平坦 99 / 斜坡 95 / 凹凸 90 / 过渡 85）
  - 平坦地形姿态稳定性 < 0.2°（Adaptive 0.16°，A* 0.10°）
  - 凹凸地形姿态稳定性 ~18°（两种算法均接近，证明姿态波动主要来自地形本身）
  - 斜坡地形：A* 姿态稳定性 6.09° 显著优于 Adaptive 的 112°，证明全局规划
    在陡峭地形上的必要性

### 7.2 不足与展望
- **7.2.1 系统不足**
  - 斜坡导航转向稳定性待提升
  - 2D 激光雷达深度信息有限
  - 无动态避障能力
- **7.2.2 改进方向**
  - 引入深度相机（RangeFinder）
  - 实现滑移补偿算法
  - 添加动态障碍物检测
  - 仿真到实物的迁移
- **7.2.3 未来展望**
  - 多机器人协同
  - 强化学习参数优化
  - 实际工程应用
